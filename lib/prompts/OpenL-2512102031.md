# Tutorial 4 Ícone do App Flutter com flutter_launcher_icons 

Este tutorial orienta a turma a configurar e gerar os ícones oficiais do aplicativo Flutter (Android, iOS, Web e Desktop) de forma padronizada e automatizada. 

# Objetivo do Tutorial 

Orientar a turma a configurar e gerar os ícones oficiais do aplicativo Flutter (Android, iOS, Web e Desktop) de forma padronizada e automatizada. 

Padronização 

Garantir que todos os ícones sigam o mesmo padrão visual em todas as plataformas 

Automatização 

Eliminar o trabalho manual de criar múltiplos tamanhos de ícones 

Multiplataforma 

Suporte completo para Android, iOS, Web e Desktop 

# Pré-requisitos 

Flutter instalado e funcionando ( flutter --version )

Projeto Flutter aberto (pasta com pubspec.yaml) 

Um ícone base em PNG, quadrado, preferencialmente 1024×1024 , sem textos e com boa margem 

Estrutura sugerida: 

assets/ icons/ app_icon.png # 1024×1024 

Dica: mantenha o arquivo simples e com contraste. No Android (Adaptive Icons) o fundo não deve ter transparência. Instalação do Pacote 

Abra o pubspec.yaml e adicione a dev-dependency e a configuração :

dev_dependencies: flutter_launcher_icons: ^0.13.0 # ou mais recente 

Por quê usar o pacote? Ele gera automaticamente todos os tamanhos e arquivos exigidos por cada plataforma, evitando erros manuais. 

# Configuração Básica 

Adicione a configuração do flutter_launcher_icons no pubspec.yaml: 

flutter_launcher_icons: platforms: android: true ios: true web: true macos: true windows: true linux: true image_path: assets/icons/app_icon.png

# Configuração Android (Adaptive Icons) 

android: "launcher_icon" adaptive_icon_background: "#FFFFFF" # cor sólida do fundo adaptive_icon_foreground: assets/icons/app_icon.png 

Os Adaptive Icons do Android permitem que o sistema aplique diferentes formas (círculo, quadrado, squircle) ao seu ícone, mantendo a consistência visual. Configuração iOS 

remove_alpha_ios: true 

Esta configuração é importante para o iOS, que não suporta transparência nos ícones de aplicativo. 

Com transparência 

Pode causar problemas de renderização no iOS 

Sem transparência 

Garante compatibilidade total com as diretrizes da Apple 

# Gerando os Ícones Limpeza e Execução

## Boas Práticas do Design do Ícone 

Tamanho base 

1024×1024 px para máxima qualidade em todos os tamanhos 

Sem texto pequeno 

Fica ilegível em tamanhos menores 

Área segura 

Deixe respiro; não encoste elementos nas bordas 

Contraste 

Garanta leitura rápida em fundos claros/escuros 

Android Adaptive: prefira foreground com transparência e background sólido .

## Alternativa Manual (se necessário) 

Android : substituir ic_launcher.png em mipmap-* e configurar adaptive via XML. Mais trabalhoso. 

iOS : substituir arquivos em AppIcon.appiconset (vários tamanhos + Contents.json). Propenso a erros. 

Web : trocar web/favicon.png e atualizar manifest.json. 

Desktop : seguir guias específicos (icns/ico e PNGs multi-tamanho). 

Recomendação: use o pacote para reduzir riscos e manter padronização entre times. 

## Problemas Comuns e Como Resolver 

"O ícone não mudou no aparelho" 

flutter clean ³ reinstale o app (desinstale/instale) 

Corte estranho no Android (círculo) 

Ajuste o foreground para caber bem no círculo; evite elementos nas bordas 

iOS com falhas de transparência 

Ative remove_alpha_ios: true 

CI/CD não gerou ícones

# Exemplo de Integração em CI (GitHub Actions) 

Ajuste para seu fluxo. A ideia é garantir que o pacote rode antes do build de release. 

name: build-android on: [push] jobs: android: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: subosito/flutter-action@v2 with: flutter-version: "stable" - run: flutter pub get - run: dart run flutter_launcher_icons - run: flutter build apk --release Perguntas Frequentes (FAQ) 

1) Posso usar SVG? 

O pacote espera um PNG final. Se tiver SVG, exporte para PNG 1024×1024. 

2) A imagem ficou serrilhada nos menores tamanhos. 

Simplifique o desenho, aumente contrastes e evite detalhes finos. Reexporte o PNG. 

3) Preciso de fundo transparente? 

Para iOS, pode ter transparência. Para Android (adaptive), use foreground com transparência e background sólido .

4) Como faço ícones diferentes por flavor (dev/prod)? 

Use múltiplas configurações (seção 6) e gere passando -c para a config desejada.
